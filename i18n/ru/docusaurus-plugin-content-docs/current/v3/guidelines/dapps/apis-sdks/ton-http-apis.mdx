# TON API на основе HTTP

Есть разные способы подключения к блокчейну TON:

1. **Поставщик данных RPC или другой API**: вам понадобится положиться на его стабильность и безопасность.
2. **[Подключение с помощью ADNL](/v3/guidelines/dapps/apis-sdks/ton-adnl-apis)**: вы подключетесь к узлу [liteserver](/v3/guidelines/nodes/running-nodes/liteserver-node). Он может иногда оказываться недоступен, но благодаря валидации, реализованной в библиотеке, он не может предоставить ложные данные.
3. **Tonlib в бинарном виде**: также подключается к лайтсерверу, так что обладает теми же преимуществами и ограничениями. Однако ваше приложение включает динамически загружаемую библиотеку, скомпилированную вовне.
4. **Только офчейн**. Такие SDK позволяют создавать и сериализовать [ячейки](/v3/documentation/data-formats/tlb/cell-boc), которые затем можно отправить с помощью API.

## Плюсы и минусы

- ✅ Лёгкость использования — идеально подходит для новичков, исследующих TON.

- ✅ Веб-ориентированность — подходит для загрузки данных из смарт-контрактов TON через веб и отправки сообщений.

- ❌ Упрощённость — не предоставляет агрегированные, исторические и индексированные датасеты; только необработанные RPC-ответы.

- ❌ Зависимость от промежуточного HTTP-слоя. Вы не сможете полностью доверять ответам сервера без [доказательств Меркла](/v3/documentation/data-formats/tlb/proofs) для проверки их подлинности.

## Узлы RPC

:::tip Состояние инфраструктуры TON

- [status.toncenter](https://status.toncenter.com/) — демонстрирует различную статистику активности узлов за последний час.

- [Tonstat.us](https://tonstat.us/) — дашборд Grafana почти в реальном времени, обновляется каждые 5 минут.
  :::

- [QuickNode](https://www.quicknode.com/chains/ton?utm_source=ton-docs) — ведущий поставщик узлов блокчейна, предлагающий быстрый доступ с интеллектуальной маршрутизацией DNS, балансировку нагрузки и масштабируемость.

- [Chainstack](https://chainstack.com/build-better-with-ton/) — предоставляет узлы RPC и индексаторы в нескольких регионах с геомаршрутизацией и балансировкой нагрузки.

- [Tatum](https://docs.tatum.io/reference/rpc-ton) — предлагает доступ к RPC-узлам TON и инструментам разработчика в простом для использования интерфейсе.

- [GetBlock](https://getblock.io/nodes/ton/) — Позволяет разработчикам подключить и протестировать свои dApps с помощью узлов GetBlock.

- [TON Access](https://www.orbs.com/ton-access/) — публичный HTTP API для The Open Network (TON).

- [TON Center](https://toncenter.com/api/v2/) — проект сообщества для быстрого доступа через API. (Получите ключ API в [@tonapibot](https://t.me/tonapibot))

- [ton-node-docker](https://github.com/fmira21/ton-node-docker) — полный узел и TON Center API в Docker-формате.

- [toncenter/ton-http-api](https://github.com/toncenter/ton-http-api) — запустите свой собственный сервис RPC API.

- [NOWNodes](https://nownodes.io/nodes) — предоставляет полноценные узлы и эксплореры через API.

- [Chainbase](https://chainbase.com/chainNetwork/TON) — API узлов и инфраструктура данных для The Open Network.

## Индексаторы

### Индекс TON Center

Индексаторы позволяют запрашивать данные о кошельках жетонов, NFT и транзакциях по определённым фильтрам, а не извлекать конкретные элементы по одному.

- Можно использовать публичный индекс TON для тестирования и разработки, [премиум-планы](https://t.me/tonapibot) для продакшена доступны на [toncenter.com/api/v3/](https://toncenter.com/api/v3/).
- Вы можете запустить свой собственный индекс TON с помощью [Worker](https://github.com/toncenter/ton-index-worker/tree/36134e7376986c5517ee65e6a1ddd54b1c76cdba) и [TON Indexer](https://github.com/toncenter/ton-indexer).

### Anton

Anton — это индексатор блокчейна The Open Network с открытым исходным кодом на Go, доступный по лицензии Apache License 2.0. Он предлагает разработчик масштабируемое, гибкое решение для доступа к данным блокчейна и их анализа. Разработчики могут также добавлять собственные смарт-контракты с собственными схемами сообщений.

- [Проект GitHub](https://github.com/tonindexer/anton) — для запуска собственного индексатора.
- [Документация по API Swagger](https://github.com/tonindexer/anton), [примеры запросов к API](https://github.com/tonindexer/anton/blob/main/docs/API.md) — помогут научиться использовать Anton.
- [Apache Superset](https://superset.apache.org/)) — для визуализации данных блокчейна.

### Узел GraphQL

Этот узел GraphQL также действует как индексатор.

- [dton.io](https://dton.io/graphql) — предоставляет данные с учётом типа контракта; также поддерживает эмуляцию транзакций и получение трейса исполнения.

## Другие API

- [TonAPI](https://docs.tonconsole.com/tonapi) — доступный API, который для упрощения пользовательского опыта абстрагирует низкоуровневые детали смарт-контрактов.
