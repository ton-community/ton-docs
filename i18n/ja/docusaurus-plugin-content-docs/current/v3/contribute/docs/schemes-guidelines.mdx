ドキュメントの一貫性を保つことは非常に重要であり、これを達成するために、スマートコントラクトのプロセスを可視化するための特定の標準が開発されました。

## グラフィック説明表記

### メッセージ処理のグラフ

メッセージ処理を描写するには、スマートコントラクトのグラフに似たグラフィカルな表現を利用するのが望ましいです。

トランザクションの順序が重要でない場合は、ラベルを省略することができます。こうすることで、ダイアグラムが簡素化され、メッセー ジや契約に関連する詳細を読みやすく、理解しやすくなります。 This simplifies the diagram, making it easier to read and understand the details related to messages and contracts.

### 書式と色

#### フォント

- 図内すべてのテキストに使用される**Inter**フォント・ファミリー。

#### 色 - ライトモード

- 鉛筆手描き（デフォルトテーマ）

#### 色 - ダークモード

- フォント `#e3e3e3`
- 背景 `#232328`
- ライトハイライト(矢印とスキームの境界線) `#058dd2`
- ダークハイライト(矢印とスキームの境界線) `#0088cc`
- 内部背景(入れ子ブロック用) `#333337`

#### バージョン管理方針

- 様々なデバイスでの読みやすさを確保するために、ドキュメント内の図をSVGフォーマットでスキームに設定する。
- 元のファイルを "/static/visio" ディレクトリの下にプロジェクトの Git リポジトリに保存すると、今後の変更が容易になります。

### 配列図

2-3アクター間の複雑で反復的な通信方式の場合は、配列図を使用することをお勧めします。 メッセージの場合は、共通の同期メッセージ矢印の表記を使用します。 For messages, use the notation of a common synchronous message arrow.
