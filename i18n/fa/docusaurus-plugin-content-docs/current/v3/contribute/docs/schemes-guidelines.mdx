## نماد توضیح گرافیکی

### نمودار پردازش پیام

برای نمایش پردازش پیام، توصیه می‌شود از یک نمایش گرافیکی شبیه به نمودار قرارداد هوشمند، همراه با برچسب‌هایی برای تراکنش‌ها و پیام‌ها استفاده کنید.

If the order of transactions doesn't matter, you can omit their labels. This simplifies the diagram, making it easier to read and understand the details related to messages and contracts.

### قالب‌ها و رنگ‌ها

#### فونت‌ها

- خانواده فونت‌های **Inter** برای تمام متن داخل نمودارها.

#### رنگ‌ها - حالت روشن

- با مداد کشیده شده (قالب پیش‌فرض)

#### رنگ‌ها - حالت تاریک

- فونت `#e3e3e3`
- پس‌زمینه `#232328`
- هایلایت روشن (فلش‌ها و حاشیه‌های الگو) `#058dd2`
- هایلایت تیره (فلش‌ها و حاشیه‌های الگو) `#0088cc`
- پس‌زمینه داخلی (برای بلوک‌های تودرتو) `#333337`

#### سیاست کنترل نسخه

- در مستندات، نمودارها را با فرمت SVG برای الگوها تنظیم کنید تا خوانایی در دستگاه‌های مختلف حفظ شود.
- فایل‌های اصلی را در مخزن Git پروژه تحت دایرکتوری "/static/visio" ذخیره کنید تا ویرایش آنها در آینده آسان‌تر باشد.

### نمودار توالی

In the case of complex and repetitive communication schemes between 2-3 actors, it is advisable to use a sequence diagram. For messages, use the notation of a common synchronous message arrow.
